# 鏈結串列（Linked List）

鏈結串列是一種 **線性資料結構（Linear Data Structure）**，由一系列 **節點（Node）** 組成，每個節點包含：

1. **資料（Data）**：節點存放的實際資訊。
2. **指標（Pointer / Next）**：指向 **下一個節點** 的連結。  
   - 在雙向鏈結串列（Doubly Linked List）中，還會有一個指向 **前一個節點** 的指標。

---

## 特點

- 節點在記憶體中 **不必連續**，透過指標串連起來。
- 適合 **頻繁插入與刪除** 的情境，尤其是中間節點操作。
- 訪問特定位置的元素需要從 **頭節點（Head）開始依次遍歷**，隨機存取效率低。
### 節點示意圖（單向鏈結串列）
```c
[Data|Next] -> [Data|Next] -> [Data|Next] -> NULL
```
- NULL 表示串列結尾。
- 第一個節點稱為 頭節點（Head）。
- 最後一個節點的 Next 指向 NULL。
# 陣列（Array） vs 鏈結串列（Linked List）比較

| 特性 | 陣列（Array） | 鏈結串列（Linked List） |
|------|----------------|-------------------------|
| **記憶體配置** | 連續記憶體 | 不連續記憶體，透過指標串連 |
| **元素存取** | O(1) 隨機存取（透過索引） | O(n) 隨機存取，需要從頭遍歷 |
| **插入/刪除效率** | 尾端插入 O(1)（若容量足夠）<br>中間插入/刪除 O(n)（需移動元素） | 頭尾或中間插入/刪除 O(1)（只需改變指標） |
| **記憶體使用效率** | 容量固定或動態增長，可能浪費空間 | 精確使用空間，但每個節點需額外存指標 |
| **快取（Cache）效率** | 高，因為連續記憶體 | 低，因為節點分散在記憶體中 |
| **大小限制** | 靜態陣列大小固定，動態陣列可增長 | 可以動態擴展，理論上只受系統記憶體限制 |
| **實作簡單性** | 簡單，索引操作直接 | 較複雜，需要指標操作 |
| **應用場景** | 適合頻繁讀取和遍歷 | 適合頻繁插入/刪除或不確定元素數量時 |

---

### 小結

- **陣列**：適合 **頻繁隨機存取** 與 **遍歷**，但中間插入/刪除代價高。  
- **鏈結串列**：適合 **靈活增刪元素**，但 **隨機存取效率低**，遍歷需從頭開始。
